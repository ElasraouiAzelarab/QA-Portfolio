## 1. Introduction

This test plan describes the strategy, scope, resources, and schedule for testing the DemoShop web application. It covers functional, integration, and regression testing for the v1.0 release.

## 2. Objectives

- Validate that all features meet the defined acceptance criteria.
- Identify defects before production deployment.
- Ensure existing functionality is not broken by new changes (regression).
- Verify the application is accessible and usable across major browsers.

## 3. Scope

### In Scope

| Feature | Priority |
|---|---|
| User Registration & Login | High |
| Product Catalog & Search | High |
| Shopping Cart | High |
| Checkout & Payment | High |
| Order History | Medium |
| User Profile Management | Medium |
| Admin Product Management | Low |

### Out of Scope

- Load / performance testing (covered in a separate plan)
- Third-party payment gateway internal logic
- Mobile native application

## 4. Test Strategy

| Testing Type | Approach |
|---|---|
| Functional Testing | Manual test case execution + automated regression suite |
| Integration Testing | API contract testing with pytest |
| UI Testing | Selenium WebDriver cross-browser suite |
| Regression Testing | Automated suite run on every PR via CI |
| Exploratory Testing | Session-based exploratory testing per sprint |

## 6. Entry & Exit Criteria

### Entry Criteria

- All user stories in the sprint are "Ready for Testing".
- Build deployed to QA environment with release notes.
- Test data and accounts provisioned.

### Exit Criteria

- 100 % of High priority test cases executed.
- Zero open Critical or High severity defects.
- Defect fix rate ≥ 95 % for Medium priority issues.
- Test execution report reviewed and signed off.

## 7. Test Deliverables

- Test cases (this repository)
- Bug reports (this repository)
- Test execution report (generated by pytest-html)
- Traceability matrix

## 8. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Environment instability | Medium | High | Dedicated QA env; smoke test before full run |
| Incomplete test data | Low | High | Automated data-seeding scripts |
| Third-party payment sandbox downtime | Medium | Medium | Mock payment gateway for automated tests |

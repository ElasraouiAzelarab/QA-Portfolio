## Severity / Priority Guide

| Severity | Definition |
|---|---|
| **Critical** | Application crash, data loss, security breach, complete feature unavailable |
| **High** | Major feature broken, no workaround available |
| **Medium** | Feature partially broken, workaround exists |
| **Low** | Cosmetic issue, minor UI inconsistency |

| Priority | Definition |
|---|---|
| **P1** | Fix immediately — blocking release or production incident |
| **P2** | Fix in current sprint |
| **P3** | Fix in next sprint |
| **P4** | Fix when convenient |

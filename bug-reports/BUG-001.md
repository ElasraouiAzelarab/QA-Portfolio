## Description

After entering valid credentials on the `/login` page and clicking the **Login** button, the application does not redirect the user to `/dashboard`. Instead, the login page reloads and the form is cleared with no error message displayed. The user is effectively stuck in a login loop.

This behaviour was introduced in build #487 and was not present in build #482.

---

## Steps to Reproduce

1. Navigate to `https://qa.demoshop.example.com/login`.
2. Enter email: `testuser@example.com`.
3. Enter password: `Test@1234`.
4. Click the **Login** button.
5. Observe the page behaviour.

---

## Expected Result

User is redirected to `https://qa.demoshop.example.com/dashboard`. The page header displays the user's name. A valid session cookie (`auth_token`) is set in the browser.

---

## Actual Result

The login page reloads. The email and password fields are cleared. No error message is shown. The URL remains `/login`. No session cookie is set (verified in DevTools → Application → Cookies).

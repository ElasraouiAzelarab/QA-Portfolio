## Description

When a valid discount coupon is applied during checkout, the discount amount is displayed as a **positive** value in the order summary (e.g., `Discount: $5.00`) instead of a **negative** value (e.g., `Discount: -$5.00`). 

Despite the misleading display, the **final order total is calculated correctly** (discount is subtracted). This is a UI display bug that could confuse users into thinking the coupon has not been applied or has added money to their total.

---

## Steps to Reproduce

1. Log in as `testuser@example.com`.
2. Add any product priced above $10.00 to the cart.
3. Navigate to `/checkout`.
4. Enter coupon code `SAVE10` in the discount field.
5. Click **Apply**.
6. Inspect the order summary section on the right side of the page.

---

## Expected Result

The discount line in the order summary displays:

```
Discount (SAVE10):    -$X.XX
```

The minus sign clearly indicates a deduction. The total correctly reflects the discount.

---

## Actual Result

The discount line displays:

```
Discount (SAVE10):    $X.XX
```

No minus sign. This appears identical to a price addition. However, the **Order Total** value is computed correctly (i.e., the discount is properly subtracted from the total).
